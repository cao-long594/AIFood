# 任务共识文档 - UI更新

## 明确的需求描述

### 功能需求
1. **日期选择UI优化**
   - 将顶部日期改为可点击的小卡片设计
   - 保留日期选择功能，点击后显示日期选择器
   - 移除日/周/月切换功能及其相关UI元素

2. **环形进度条实现**
   - 在紫色卡片中实现环形进度条
   - 环形进度条根据当天卡路里摄入量与目标卡路里的比值动态填充
   - 显示完成百分比

### 非功能需求
- UI设计应简洁美观，符合应用整体风格
- 进度条动画应流畅自然
- 代码结构清晰，易于维护

## 技术实现方案

### UI实现
- 修改fragment_home.xml布局文件
- 使用CardView实现日期小卡片
- 使用两个ProgressBar叠加实现环形进度条效果：
  - 底层ProgressBar作为背景圆环
  - 上层ProgressBar显示实际进度
- 设置合适的颜色和样式

### 逻辑实现
- 修改HomeFragment.java
- 移除日/周/月切换相关代码
- 添加环形进度条更新逻辑
- 计算卡路里完成百分比并更新进度条

### 技术约束
- 仅使用Android原生组件，不引入第三方图表库
- 兼容Android 5.0及以上版本
- 遵循项目现有的代码规范

## 任务边界限制

### 包含内容
- 修改fragment_home.xml布局文件
- 修改HomeFragment.java文件
- 添加必要的注释和文档

### 不包含内容
- 修改其他Fragment或Activity
- 添加新的功能模块
- 优化现有功能的算法

## 验收标准

1. **UI展示**
   - 日期显示区域为小卡片样式
   - 无日/周/月切换按钮
   - 紫色卡片中显示环形进度条

2. **功能验证**
   - 点击日期卡片能正常弹出日期选择器
   - 环形进度条根据卡路里完成度正确填充
   - 进度百分比显示准确

3. **代码质量**
   - 代码结构清晰，注释完整
   - 无编译错误和警告
   - 遵循项目代码规范
